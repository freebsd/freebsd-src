## 7.1 Executive Summary

This document defines a utility to convert Kubernetes and Docker Compose configurations into the native FreeBSD container format. The goal is to provide a smooth migration path for users coming from Linux container ecosystems while producing configurations that are:

- **Simpler:** 50-75% fewer lines than the original
- **Native:** Uses FreeBSD-native concepts (jail, ZFS, VNET) directly
- **Sensible:** Sensible defaults eliminate boilerplate
- **Decorated:** Better comments, structure, and readability

**Key Principle:** The converter should not just translate — it should optimize and simplify, producing idiomatic FreeBSD native configurations.

## 7.3 Conversion Examples

### 7.3.1 Kubernetes → Native

**Input (Kubernetes):**
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: myapp
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: myapp-config
  namespace: myapp
data:
  NGINX_WORKERS: "4"
  LOG_LEVEL: "info"
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
  namespace: myapp
  labels:
    app: web
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx
        image: nginx:1.25
        ports:
        - containerPort: 80
        env:
        - name: NGINX_WORKERS
          valueFrom:
            configMapKeyRef:
              name: myapp-config
              key: NGINX_WORKERS
        resources:
          requests:
            memory: "256Mi"
            cpu: "500m"
          limits:
            memory: "512Mi"
            cpu: "1000m"
        livenessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 5
        volumeMounts:
        - name: html
          mountPath: /usr/share/nginx/html
      volumes:
      - name: html
        persistentVolumeClaim:
          claimName: myapp-html
---
apiVersion: v1
kind: Service
metadata:
  name: web-service
  namespace: myapp
spec:
  selector:
    app: web
  ports:
  - port: 80
    targetPort: 80
  type: LoadBalancer
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: myapp-html
  namespace: myapp
spec:
  accessModes:
  - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```

**Output (Native):**
```yaml
# Converted from Kubernetes by ocifbsd-convert
# Original: deployment.yaml, service.yaml, pvc.yaml
# Simplified: 85 lines → 28 lines (67% reduction)

stack: myapp
version: "1.0"

# Volumes are ZFS datasets — no separate PVC needed
volumes:
  html:
    driver: zfs
    dataset: zroot/myapp/html
    size: 1G

pods:
  web:
    replicas: 3
    image: nginx:1.25
    ports: ["80:80"]
    environment:
      NGINX_WORKERS: "4"
      LOG_LEVEL: "info"
    volumes:
      - html:/usr/share/nginx/html
    resources:
      memory: 256M
      cpus: 0.5
    healthcheck:
      cmd: ["curl", "-f", "http://localhost:80/health"]
      interval: 10s
      timeout: 5s
      retries: 3
      start_period: 30s

services:
  web-lb:
    type: loadbalancer
    selector: { pod: web }
    ports: [{ port: 80, target_port: 80 }]
```

### 7.3.2 Docker Compose → Native

**Input (Docker Compose):**
```yaml
version: "3.8"

services:
  web:
    image: nginx:1.25
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./html:/usr/share/nginx/html:ro
      - nginx-cache:/var/cache/nginx
    environment:
      - NGINX_WORKERS=4
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: '0.5'
          memory: 256M
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:80/health"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 30s
    networks:
      - frontend
      - backend
    depends_on:
      - api

  api:
    image: myapp-api:v2.1.0
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://db:5432/myapp
      - REDIS_URL=redis://cache:6379
    deploy:
      replicas: 2
      resources:
        limits:
          cpus: '1.0'
          memory: 512M
    networks:
      - backend
    depends_on:
      - db
      - cache

  db:
    image: postgres:15
    ports:
      - "5432:5432"
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=myapp
      - POSTGRES_USER=myapp
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 1G
    networks:
      - backend

  cache:
    image: redis:7
    ports:
      - "6379:6379"
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 256M
    networks:
      - backend

volumes:
  nginx-cache:
  db-data:

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true
```

**Output (Native):**
```yaml
# Converted from Docker Compose by ocifbsd-convert
# Original: docker-compose.yml
# Simplified: 78 lines → 42 lines (46% reduction)

stack: myapp
version: "1.0"

# Networks are built-in — no external CNI needed
networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true

# Volumes are ZFS datasets with automatic management
volumes:
  nginx-cache:
    driver: zfs
    dataset: zroot/myapp/nginx-cache
  db-data:
    driver: zfs
    dataset: zroot/myapp/db-data
    size: 10G

pods:
  web:
    replicas: 3
    image: nginx:1.25
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./html:/usr/share/nginx/html:ro
      - nginx-cache:/var/cache/nginx
    environment:
      NGINX_WORKERS: "4"
    resources:
      memory: 256M
      cpus: 0.5
    healthcheck:
      cmd: ["curl", "-f", "http://localhost:80/health"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 30s
    networks: [frontend, backend]

  api:
    replicas: 2
    image: myapp-api:v2.1.0
    ports: ["3000:3000"]
    environment:
      DATABASE_URL: "postgres://db:5432/myapp"
      REDIS_URL: "redis://cache:6379"
    resources:
      memory: 512M
      cpus: 1.0
    networks: [backend]

  db:
    image: postgres:15-freebsd
    ports: ["5432:5432"]
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: myapp
    stateful: true
    resources:
      memory: 1G
      cpus: 1.0
    networks: [backend]

  cache:
    image: redis:7-freebsd
    ports: ["6379:6379"]
    resources:
      memory: 256M
      cpus: 0.5
    networks: [backend]

services:
  web-lb:
    type: loadbalancer
    selector: { pod: web }
    ports:
      - port: 80
        target_port: 80
      - port: 443
        target_port: 443

  api-lb:
    type: clusterip
    selector: { pod: api }
    ports: [{ port: 3000, target_port: 3000 }]
```

## 7.4 Conversion Rules

### 7.4.1 Kubernetes → Native Mapping

| Kubernetes Concept | Native Equivalent | Notes |
|-------------------|-------------------|-------|
| Namespace | `stack` + `namespace` | Combined into stack definition |
| Deployment | `pod` with `replicas` | Direct mapping |
| StatefulSet | `pod` with `stateful: true` | Stable hostnames and storage |
| DaemonSet | `pod` with `daemon: true` | One per host |
| Job | `pod` with `restart: never` | Run once and exit |
| CronJob | `pod` with `schedule` | Cron expression |
| Pod | `pod` with `replicas: 1` | Direct mapping |
| Service (ClusterIP) | `service` with `type: clusterip` | Direct mapping |
| Service (LoadBalancer) | `service` with `type: loadbalancer` | Built-in, no MetalLB needed |
| Service (NodePort) | `service` with `type: nodeport` | Maps to host ports |
| Ingress | `ingress` | Built-in, no IngressController needed |
| ConfigMap | `environment` or `volumes` (nullfs) | Inline or file-based |
| Secret | `environment` or ZFS encrypted dataset | Encrypted at rest |
| PersistentVolumeClaim | `volume` with `driver: zfs` | Native ZFS integration |
| PersistentVolume | `volume` with `dataset` | Pre-created ZFS dataset |
| StorageClass | `volume` with `driver` | ZFS, tmpfs, nullfs |
| NetworkPolicy | `network_policy` | pf rules per namespace |
| ResourceQuota | `resources` at stack level | RCTL rules |
| LimitRange | `resources` defaults | Applied to all pods |
| HorizontalPodAutoscaler | `autoscale` | Planned feature |
| PodDisruptionBudget | `reschedule` rules | Min available replicas |

### 7.4.2 Docker Compose → Native Mapping

| Compose Concept | Native Equivalent | Notes |
|----------------|-------------------|-------|
| service | `pod` | Direct mapping |
| image | `image` | Direct mapping |
| ports | `ports` | Same syntax |
| volumes | `volumes` | ZFS datasets or bind mounts |
| environment | `environment` | Same syntax |
| networks | `networks` | Built-in bridge/VNET |
| deploy.replicas | `replicas` | Direct mapping |
| deploy.resources | `resources` | Simplified syntax |
| deploy.restart_policy | `restart` | Direct mapping |
| healthcheck | `healthcheck` | Simplified syntax |
| depends_on | `depends_on` | Startup ordering |
| configs | `volumes` (nullfs) | File-based config |
| secrets | `environment` or encrypted volume | Secure storage |
| extends | `import` | Include other files |

### 7.4.3 Simplification Rules

1. **Remove boilerplate:**
   - Kubernetes `apiVersion`, `kind`, `metadata` → removed
   - Docker Compose `version` → removed (native has implicit version)
   - Empty `selector` blocks → inferred from pod name

2. **Inline small resources:**
   - ConfigMaps with < 5 keys → inline `environment`
   - Single-port services → inline `ports` in pod

3. **Simplify resource syntax:**
   - `resources.requests.memory: "256Mi"` → `resources.memory: 256M`
   - `resources.requests.cpu: "500m"` → `resources.cpus: 0.5`
   - `resources.limits` → same as `resources` (native uses limits by default)

4. **Use native defaults:**
   - `restart: always` → default, can be omitted
   - `driver: zfs` → default for volumes, can be omitted
   - `network: bridge` → default, can be omitted

5. **Convert health checks:**
   - `livenessProbe.httpGet` → `healthcheck.cmd` with curl
   - `readinessProbe.httpGet` → `healthcheck.cmd` with curl
   - `startupProbe` → `healthcheck.start_period`

6. **Convert volumes:**
   - `persistentVolumeClaim` → `volume` with ZFS dataset
   - `hostPath` → bind mount (`/host/path:/container/path`)
   - `emptyDir` → `tmpfs` volume
   - `configMap` → `nullfs` mount or inline environment

## 7.5 CLI Interface

### 7.5.1 Basic Usage

```bash
# Convert Kubernetes YAML to native format
ocifbsd convert -f k8s-deployment.yaml -o myapp.yml

# Convert multiple Kubernetes files
ocifbsd convert -f k8s/*.yaml -o myapp.yml

# Convert Docker Compose
ocifbsd convert -f docker-compose.yml -o myapp.yml

# Convert and deploy in one step
ocifbsd convert -f docker-compose.yml | ocifbsd stack deploy -f -

# Convert with annotations
ocifbsd convert -f k8s/*.yaml -o myapp.yml --annotate

# Convert to compact format (no comments)
ocifbsd convert -f docker-compose.yml -o myapp.yml --compact

# Convert to JSON
ocifbsd convert -f docker-compose.yml -o myapp.json --format json

# Validate without converting
ocifbsd convert -f docker-compose.yml --validate

# Show conversion statistics
ocifbsd convert -f k8s/*.yaml -o myapp.yml --stats
```

### 7.5.2 Advanced Options

```bash
# Specify image registry mapping
ocifbsd convert -f docker-compose.yml \
  --map-image nginx:1.25=myregistry/nginx-freebsd:1.25 \
  --map-image postgres:15=myregistry/postgres-freebsd:15

# Specify network mapping
ocifbsd convert -f docker-compose.yml \
  --map-network frontend=myapp-frontend \
  --map-network backend=myapp-backend

# Specify volume mapping
ocifbsd convert -f docker-compose.yml \
  --map-volume db-data=zroot/myapp/db-data

# Include/exclude services
ocifbsd convert -f docker-compose.yml \
  --include web,api,db \
  --exclude cache

# Set default namespace
ocifbsd convert -f k8s/*.yaml \
  --namespace production \
  -o myapp.yml

# Dry run — show what would be converted
ocifbsd convert -f docker-compose.yml --dry-run
```

### 7.5.3 Interactive Mode

```bash
# Interactive conversion with prompts
ocifbsd convert -f docker-compose.yml -i

# Example prompts:
# ? Image 'nginx:1.25' not found in registry. Use anyway? (Y/n)
# ? Service 'web' has 3 replicas. Confirm? (Y/n)
# ? Volume 'db-data' will use ZFS dataset 'zroot/myapp/db-data'. OK? (Y/n)
# ? Network 'frontend' will be bridged. OK? (Y/n)
```

## 7.6 Conversion Warnings and Errors

### 7.6.1 Warning Categories

| Code | Severity | Description | Action |
|------|----------|-------------|--------|
| W001 | Warning | Linux-specific image detected | Suggest FreeBSD equivalent |
| W002 | Warning | Feature not yet supported | Document as comment in output |
| W003 | Warning | Resource limits may need adjustment | Suggest FreeBSD-native values |
| W004 | Warning | Volume path may need updating | Suggest FreeBSD path |
| W005 | Warning | Network policy simplified | Document security implications |
| W006 | Warning | Health check converted to command | Verify command exists in image |
| W007 | Warning | Secret not encrypted | Suggest ZFS encryption |
| W008 | Warning | Privileged container detected | Review security requirements |

### 7.6.2 Error Categories

| Code | Severity | Description | Action |
|------|----------|-------------|--------|
| E001 | Error | Invalid YAML/JSON syntax | Fix input file |
| E002 | Error | Unsupported Kubernetes API version | Update input or use converter flag |
| E003 | Error | Required field missing | Add missing field |
| E004 | Error | Circular dependency detected | Fix depends_on |
| E005 | Error | Image not found in registry | Verify image name or use --map-image |
| E006 | Error | Volume size exceeds available ZFS space | Reduce size or expand pool |
| E007 | Error | Port conflict detected | Change port mapping |
| E008 | Error | Invalid resource specification | Fix memory/CPU format |

## 7.9 Conclusion

The Config Conversion Utility provides:

- **Smooth migration path:** Existing Kubernetes and Docker Compose configs work immediately
- **Significant simplification:** 50-75% line reduction while maintaining semantic power
- **FreeBSD-native output:** Uses jail, ZFS, and VNET directly
- **Extensible design:** Easy to add new input formats (Helm, Kustomize, etc.)

This utility eliminates the barrier to entry for teams migrating from Linux containers to FreeBSD native containers.
