### Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Runtime approach | Native `ocifbsd` | Avoids Linux emulation overhead |
| Storage driver | ZFS snapshots/clones | Native, efficient, deduplicated |
| Networking | VNET + bridge + pf | Native FreeBSD stack |
| Resource limits | RCTL | Native FreeBSD resource control |
| Security | MAC (mac_biba, mac_lomac, mac_mls) | Native FreeBSD mandatory access control |
| Orchestration | Built-in, lightweight | Avoids Kubernetes complexity |
| Config format | Enhanced YAML/JSON | Simpler than Kubernetes YAML |
